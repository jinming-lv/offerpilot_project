# 一、项目定位（答辩第一页）

## 项目背景

随着大模型的发展，求职者面临以下问题：

* 简历不知道如何优化
* 岗位匹配效率低
* 不知道自己的能力差距
* 缺少针对性的学习建议
* 面试准备没有方向

目前ChatGPT只能回答单个问题，**无法完成完整的求职流程**。

因此，我们设计一个**基于多Agent的大模型智能求职平台**。

---

## 项目目标

实现一个完整的求职闭环：

```text
上传简历

↓

岗位匹配

↓

能力分析

↓

学习建议

↓

模拟面试

↓

生成求职报告
```

最终目标：

> **让Agent像职业规划师一样帮助求职。**

# 八、答辩演示流程（建议控制在 5 分钟）

准备一份示例简历（例如应届 Python 后端开发）和一个岗位 JD。

演示顺序：

1. 上传简历
2. 输入目标岗位（Python 后端开发）
3. 系统自动解析简历
4. 展示岗位匹配度（雷达图 + 分数）
5. 展示能力差距（Redis、Docker 等）
6. 点击"开始模拟面试"，回答一两个问题
7. 展示学习路线（14 天计划）
8. 一键生成《求职分析报告》

这样可以完整展示 **多 Agent 协作**，每一步都有可视化结果，非常适合生产实习答辩。

# 九、我建议的 MVP（3 天内能高质量完成）

为了控制风险，我建议**不要做太多花哨功能**，而是把下面 6 个功能做到稳定：

* ✅ 上传 PDF/Word 简历并解析
* ✅ 输入岗位 JD，自动计算匹配度
* ✅ 生成能力画像（技能标签 + 雷达图）
* ✅ 自动生成 5–10 道针对性面试题
* ✅ 输出个性化学习路线（7 天或 14 天）
* ✅ 一键生成求职分析报告

这样既完整体现了 **Agent + RAG + 大模型** 的能力，又能保证 **4 个人 3 天** 内完成一个完成度较高、演示效果好的课程项目。
